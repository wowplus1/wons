## 개선 내용
1. **컬럼 전면 재배치**: 사용자가 정의한 17개 항목 순서(`체크박스 / No / 의뢰일 / 구분 / 세공 요청사항 / 모델 / 재질 / 색상 / 스톤세팅스펙 / 구매단가 / 공임단가 / 알수합계 / 수량 / 중량(금) / 중량(스톤) / 단계별 세공 무게 / 되돌리기`)로 테이블 레이아웃을 개편하였습니다.
2. **구분 필드 추가**: 주문 품목의 `division` 속성을 시각적인 상태 배지(판매, 결제, 반품, DC)와 함께 렌더링하도록 반영했습니다.
3. **공임 및 구매 단가 정밀 연계**:
   - 기존의 단일 공임비 열을 **구매단가**(`labor_cost`)와 **공임단가**(`labor_base` / `labor_extra` / `labor_main` / `labor_sub` 슬래시구분 표시)로 분할하여 한눈에 확인할 수 있도록 상세화했습니다.
   - 단가 노출은 테이블 상단의 "인쇄 시 공임비/단가 표시"(`showPrice`) 스위치 상태에 따라 동적으로 노출/숨김 전환이 됩니다.
4. **수량 및 중량 계산식 적용**:
   - **알수합계**: 메인 스톤 개수와 보조 스톤 개수를 합산하여 표시합니다.
   - **중량(금)**: `item.gold_weight` 데이터를 바인딩하여 출력합니다.
   - **중량(스톤)**: 메인 스톤 실중량 합계 및 보조 스톤의 마스터 캐럿 중량(`weight_carat`)을 역추적해 더한 `stonesWeightTotal` 값을 산출하여 출력합니다.
5. **실무 사용 편의성 보완**: 거래처 열을 별도 열로 유지하지 않는 대신, 모델 번호 하단에 회색 폰트로 거래처명(`customerName`)을 작게 표기하여 어떤 업체의 작업 품목인지 빠르게 식별할 수 있도록 배려했습니다.
6. **확인을 위한 조치**: 사용자가 직접 화면을 확인할 수 있도록 로컬 개발 서버(`npm run dev`) 구동을 개시하였습니다.
7. **구매단가 합산 공식 보완**: 사용자의 요구사항에 맞추어 모델 구매원가와 중심/보조 스톤 구매원가 합계를 실시간 연산하여 최종 구매단가에 반영하였습니다.
8. **구매단가 산식 오차 수정 (2차 보완)**: 벨R-581 모델의 구매단가가 `42,000원`으로 노출되는 오차(기대값 `41,500원`)를 교정하기 위해, 스톤 구매단가 합산 시 주문서의 가변 수량이 아닌 **카탈로그 상품 자체에 등록된 모든 기본 스톤 스펙(`default_stones`)의 고정 합계**를 기준으로 연산하도록 산출 방식을 개편하였습니다.
9. **모델번호 대소문자/공백 제거 비교 보완 (3차 보완)**: 카탈로그의 '벨R-581' 모델을 찾을 때 미세한 공백 차이로 매칭이 실패하여 주문서의 잘못 입력된 금액인 `25,500원`이 보정되지 않고 그대로 사용되어 42,000원으로 출력된 현상을 파악했습니다. 모델번호 비교 시 대소문자 무시 및 공백 제거(`.toUpperCase().trim()`)를 적용하여 카탈로그의 정밀 정보가 연동되도록 개선하였습니다.
10. **중간 공백 제거 비교 보완 (4차 보완)**: 기존 `.trim()` 방식은 문자열 앞뒤 공백만 지워주므로, 모델명 중간의 공백(예: `벨 R-581` 과 `벨R-581`)이 존재하는 경우 매칭 실패가 유지됨을 진단했습니다. 모든 공백을 강제로 제거하는 정규식 `.replace(/\s+/g, '')` 비교 공식을 도입하여 카탈로그 데이터가 완벽하게 바인딩되도록 개선하였습니다.
11. **구매단가 세부 연산 추적 기능 추가 (5차 보완)**: 모델 공백 수정 후에도 여전히 `42,000원`이 표시되는 문제를 추적하기 위해, 테이블의 **구매단가** 셀 내부에 실시간 연산 디버깅 정보(매칭 여부, 연산된 모델 구매가 및 스톤 구매가 구성)를 노출하도록 수정하였습니다.
12. **스톤 마스터 상세 추적 추가 (6차 보완)**: 메인 스톤 단가가 `1,000원`으로 잡히는 오차 원인을 마스터 수준에서 정밀하게 파악하기 위해, 디버그 정보에 물려진 스톤의 **ID와 이름**을 연동하여 추가 노출하도록 수정하였습니다.
13. **카탈로그 바인딩 단가 하드코딩 버그 수정 (7차 보완)**: 디버그 역추적을 통해 메인 스톤(`메인/RD/1부물림3.0`)의 스톤 마스터 상 실제 구매단가가 `1,000원`으로 등록되어 있음을 확인했습니다. 그러나 카탈로그 등록/수정 화면(`CatalogRegisterForm.tsx`)에서 수정 로드 시 스톤 단가를 실제 마스터 단가가 아닌 무조건 `500원`으로 화면에 하드코딩하여 바인딩하는 버그를 발견하여, 이를 마스터 값 연계 방식으로 정상 교정하였습니다.
