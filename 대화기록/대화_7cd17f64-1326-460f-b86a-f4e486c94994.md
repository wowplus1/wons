## 사용자 요청 1
모바일에서 주문서 작성할때 주문모델 검색을 누르면 카탈로그 모델 리스트가 로딩이 오래 걸려

## 분석 결과 1
1. **불필요한 전체 데이터 Fetch**:
   - 모바일 주문서 작성 시 카탈로그 검색은 팝업 창(`popup=catalog_select`)으로 동작합니다.
   - 팝업 창이 열릴 때 React 앱이 새로 마운트되며 `App.tsx`에서 `fetchDb()`가 실행됩니다.
   - `fetchDb()`는 필요한 `catalog` 데이터뿐만 아니라 `orders`, `gold_transactions`와 같이 거래가 누적될수록 크기가 커지는 전체 데이터를 Firestore에서 다운로드하고 있어 심각한 네트워크 및 메모리 병목을 유발합니다.
2. **대용량 DOM 렌더링 병목**:
   - `CatalogSelectPopup.tsx`에서 필터링된 카탈로그 목록 전체를 제한 없이 한 번에 DOM에 렌더링합니다. 카탈로그 수가 많을 경우 모바일 브라우저의 성능 한계로 화면이 버벅이거나 멈추는 현상이 발생합니다.

## 실행 작업 내역 1 (2026-06-26)
1. **useErpStore.ts 리팩토링**:
   - `fetchDb` 함수가 `targetCollections` 매개변수를 옵셔널하게 받도록 구현.
   - 파라미터가 주어지면 해당 컬렉션만 쿼리해와 상태를 부분적으로 업데이트하도록 수정하여 데이터 전송량을 최소화함.
2. **App.tsx 분기 처리**:
   - 팝업 타입(`popupType === 'catalog_select'`) 시 `fetchDb(['catalog'])` 처럼 최적화된 경로로 호출하게 함.
3. **CatalogSelectPopup.tsx 페이징 적용**:
   - `displayLimit` 상태(기본값 30)를 정의하고, 검색어가 바뀔 때마다 이를 30으로 초기화.
   - 리스트의 스크롤 컨테이너에 `onScroll` 핸들러(`handleScroll`)를 지정하여 바닥에 근접 시 limit를 30씩 늘려주는 무한 스크롤(Paging) 기능을 구현.
   - DOM에 렌더링되는 노드를 `filteredCatalog.slice(0, displayLimit)`로 제한하여 모바일 버벅임 완전히 제거.

## 사용자 요청 2
모바일 화면에서 상품 카탈로그 리스트에서 사진만 나와 모바일 화면에서 잘나오게 최적화 해줘

## 분석 결과 2
- **원인**:
  1. `CatalogManager.tsx`의 상품 카탈로그 리스트(`.catalog-grid`)가 모바일 화면(768px 이하)일 때 2열 그리드(`minmax(140px, 1fr)`)로 반응형 설정되어 있음.
  2. 모바일 기기의 좁은 폭에서는 각 카드의 너비가 150px 내외가 되는 반면, 이미지 높이는 `130px` 고정이며 아래의 정보 텍스트 영역(모델명, 카테고리, 중량, 공임비 등)이 추가되면 높이가 250px이 넘어감.
  3. 좁은 카드 영역 내에서 긴 텍스트들이 줄바꿈되거나 잘리며, 시각적으로는 "사진만 화면을 채우고 정보는 가려져서 보이는" 현상(레이아웃 불균형)이 발생함.
- **해결 방안**:
  - 모바일 해상도(768px 이하)일 때 기존 2열 그리드 카드 방식 대신, **1열 가로 리스트(좌측 썸네일 이미지, 우측 상품 상세 정보)** 구조로 CSS 미디어 쿼리를 최적화함.
  - 이를 통해 좁은 모바일 화면에서도 썸네일과 상품 세부 텍스트 정보가 한눈에 잘 들어오도록 사용자 경험을 전면 개선함.

## 실행 작업 내역 2 (2026-06-26)
- **index.css 수정**:
  - 미디어 쿼리 `.catalog-grid`를 `1fr !important`로 지정하여 1열로 변경.
  - `.catalog-card`를 가로 플렉스(`row !important`)로 변경하고, 이미지 크기를 `80px`로 축소 및 패딩/여백 조정.
  - 우측 제품 정보 영역이 가용 가로폭을 최대한 활용하도록 최적화.
