## 분석 결과 6
- **원인**:
  1. 배치 파일이 UTF-8 인코딩으로 저장되어 Windows CMD의 기본 인코딩인 ANSI(cp949)와 호환되지 않아 한글이 깨졌습니다.
  2. 줄바꿈 문자가 Unix 형식(LF)인 경우, CMD에서 여러 행이 한 행으로 인식되어 구문 오류(`'mmit)...'` 등)가 발생하는 원인이 됩니다.
- **해결 방안**:
  - 배치 파일의 메시지 텍스트를 영문으로 직관적이게 수정하여 한글 깨짐의 여지를 차단했습니다.
  - Windows 호환을 위해 줄바꿈 형식을 `CRLF`로 강제 변환하고 파일 인코딩을 `cp949(ANSI)`로 작성하는 파이썬 헬퍼 스크립트(`create_bat.py`)를 이용해 배치 파일을 재생성했습니다.
