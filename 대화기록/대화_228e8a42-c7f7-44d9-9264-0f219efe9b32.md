  - 원인은 두 가지일 수 있습니다:
    1. 데이터베이스에서 로드한 개별 공임 객체(`lf`)의 단가 필드(`grade_1` 등)가 문자열로 취급되어 필터링(`+` 합산 연산) 시 참으로 유도되었으나, 실제 렌더링할 때 문자열과 0의 판단 과정에서 예외가 발생했거나,
    2. 부모 컨테이너가 `display: flex; flex-direction: column`이며 높이 제한(`maxHeight: 180px`)이 있고 내부 요소들의 축소율(`flex-shrink`) 때문에 테이블 바디 영역이 0px로 찌그러져 짤렸을 가능성.
- **해결 방안**:
  - `Number` 형변환을 각 필드에 안전하게 걸어 숫자 합산이 명확히 처리되도록 필터와 매핑 로직을 정교화합니다.
  - 테이블 래퍼 `div`에 `flexShrink: 0`을 지정하여 flex 부모의 높이 제한 상황에서도 내용이 찌그러져서 텍스트가 짤리는 현상을 원천 방지합니다.
  - 실시간 데이터 검증을 위해 각 표 블록 하단에 `{JSON.stringify(lf)}`를 출력해주는 디버그 문자열을 임시로 렌더링합니다.
