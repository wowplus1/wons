# ERP 카탈로그 등록 폼 개선 대화 로그

- **일시**: 2026-06-30
- **작업 내용 요약**:

## 1차 개선 작업
1. 글씨 및 회색 가시성 강화 (CSS 테마 변수 연동)
2. 스톤 선택 시 해당되는 개당중량, 구매단가(스톤 마스터 purchase_price 연동), 등급별 금액 자동 세팅
3. 스톤 입력을 최대 4개로 제한하고 하단에 합계 정보 배치
4. 알수 소계, 중량 소계, 수동차감중량, 총 차감중량, 최종 금 중량이 계층 구조로 명확히 표시되는 요약 세로바 구축
5. 숫자 입력 시 0 자동 소거 및 복원 UX 적용
6. 공임 및 원가 설정 입력란 개선
7. 입력 필드 전체의 테두리 및 배경 색상 선명화 (var 테마 연동 및 하드코딩 rgba 탈피)
8. 스톤 자동완성 검색 기능 도입 (드롭다운 자동 닫힘 및 텍스트 시인성 확보)
9. 전체 Vite 프로덕션 빌드 성공 및 검증 완료

## 2차 개선 작업
- **요청 사항**: 카달로그 상세 화면에서 관련제품 목록에 관련제품이 일치하는 것만 나오도록 수정
- **개선 내용**: 
  1. 기존의 썸네일 14개 강제 복제 렌더링 루프를 제거.
  2. 세트 관련 모델 번호(`set_model_numbers`)와 정확히 매칭되는 일치 카탈로그 정보만 중복 없이 1:1로 렌더링되도록 수정.

## 3차 개선 작업
- **요청 사항**: 카달로그 상세 화면에서 노출되는 공임비 테이블의 색상이 실제 등록된 색상 정보와 불일치함
- **개선 내용**:
  1. 상세 페이지 공임비 분석 테이블 헤더에 기존에 `G` 로 하드코딩되어 강제로 보여주던 항목을 제거.
  2. 해당 모델의 재질별 실제 기본 공임에 설정된 색상(`baseFee.color` / `materialLaborMap.color`)을 동적으로 연동하여, `W` 나 `P`, `전체` 등으로 등록된 원래의 색상이 정확하게 노출되도록 보안.

## 4차 개선 작업
- **요청 사항**: 카달로그 상세 화면에 알 공임비(추가공임)가 안 들어가 있음 (0원으로 출력됨)
- **개선 내용**:
  1. 기존 카탈로그 등록/수정 화면(`CatalogRegisterForm.tsx`)에서 알 공임비(추가공임)를 적는 인풋창이 아예 누락되어 무조건 `extra_labor_fee: 0` 으로 하드코딩되어 저장되는 원인을 찾아냄.
  2. 등록 화면 상단 사양 영역에 **알 공임비(개당)** 입력 필드를 새로 추가하여 `extraLaborFee` state를 신설.
  3. 카탈로그 저장 시 `extra_labor_fee: parseFloat(extraLaborFee) || 0` 형태로 제대로 값이 저장되도록 수정.
  4. 수정 모드 바인딩 시에도 `extra_labor_fee`를 정상적으로 반환 및 바인딩되도록 조치.
  5. 이를 통해 상세 화면에서 보조 스톤 알 수 세팅에 따른 알 공임비 및 합계 금액이 정상적으로 연산 및 출력되도록 해결.

## 5차 개선 작업
- **요청 사항**: 알 공임비를 입력하고 저장했으나 상세 화면에서 여전히 '알' 공임이 0원으로 출력되는 버그
- **개선 내용**:
  1. 원인 분석 결과, 사용자가 스톤 검색창에 스톤 명칭(예: 다이아)을 직접 타이핑만 하고 드롭다운 목록의 항목을 클릭하여 선택하지 않는 경우, 내부적으로 `stone_id`가 지정되지 않아 저장(save) 시 스톤 데이터가 아예 누락되어 중량 및 수량이 0으로 저장되는 UX 충돌을 파악.
  2. 저장 시점에 사용자가 타이핑한 텍스트(`search_term`)와 일치하는 스톤 정보가 있을 시 자동으로 `stone_id`를 매핑해주는 **안전 보정 장치(Auto Matcher)**를 `defaultStonesMap` 생성 로직에 이식.
  3. 드롭다운 선택 누락 시에도 텍스트가 정확하다면 정상 매칭 및 저장되도록 보완하여, 알 수와 알 공임비가 상세 페이지에 문제없이 반영되도록 해결.

## 6차 개선 작업
- **요청 사항**: 상세 페이지 공임 분석의 '알' 공임비가 단일 세팅비(extra_labor_fee)로만 기산되어 실제 스톤 테이블에 지정된 등급별 공임비 합산액이 누락되는 현상
- **개선 내용**:
  1. 기존에 모든 등급에 동일하게 `stoneQty * item.extra_labor_fee` 만 뿌려주던 단선적인 연산 구조를 변경.
  2. 각 등급(g등급 = 1, 2, 3, 4등)을 매핑 돌면서, 해당 카탈로그 아이템에 세팅된 스톤들의 **해당 등급별 스톤 공임비 합산금액**(`matchedStone.grade_prices[grade_g] * quantity`)을 실시간으로 집계하는 `gradeStoneLaborSum` 연산 적용.
  3. 등급별 스톤 공임 합산에 기본 알물림 세팅비(`stoneQty * extra_labor_fee`)를 가산한 값을 `gradeStoneLaborTotal` 로 정의하여, 공임 분석 테이블의 '알' 및 '합계/시세' 가격이 등급별로 완벽하게 다르게 연동되도록 조치 완료.

## 7차 개선 작업
- **요청 사항**: 카달로그 상세 화면의 공임 분석 테이블에서 '시세' 칸에 금값이 포함되지 않고 공임 합계만 나오는 현상
- **개선 내용**:
  1. 스토어의 실시간 금 시세 데이터(`currentRates`/`goldRates`)를 동적 수신하여 해당 상품의 기본 재질(14K, 18K, 24K 등)에 대응하는 1g당 판매 시세를 결정.
  2. 상세 화면에 출력되는 해당 상품의 **순수 금중량**(`pureGoldWeight`)에 당일 금 시세를 곱하여 실시간 금값(`goldCost`)을 산출.
  3. 테이블의 '시세' 칸을 기존의 공임 합계(`totalCost`)에서 **공임 합계 + 금값**(`totalCost + goldCost`)으로 전격 개조하여, 금값이 온전히 가산된 통합 판매 가격이 정확히 표기되도록 완성.

## 8차 개선 작업
- **요청 사항**: 카달로그 상세화면의 스톤 사양 테이블 내 '비고' 항목이 실제 스톤 비고(설명)를 출력하지 못하고 하드코딩된 단가 텍스트를 출력하는 문제
- **개선 내용**:
  1. 카탈로그 데이터 모델(`CatalogItem.default_stones`) 구조에 `description?: string` 필드를 신규 추가.
  2. 등록 화면(`CatalogRegisterForm.tsx`)에서 상품 저장 및 수정 모드 로드 시, 사용자가 직접 기입한 **스톤 설명(description)** 데이터가 유실되지 않고 안전하게 바인딩되어 데이터베이스에 기록되도록 연동.
  3. 상세 화면(`CatalogDetailView.tsx`)에서 스톤의 비고란을 기존의 하드코딩된 보조 세팅비 텍스트 대신 `ds.description` 필드를 동적으로 출력하도록 전면 개편.
  4. 비고란이 비어 있는 경우 공란으로 깔끔하게 비우며, 기존에 꼬리에 강제로 추가 렌더링되던 불필요한 빈 `기타:` 행 구조를 깔끔하게 제거.

## 9차 개선 작업
- **요청 사항**: 카탈로그 상세 화면 하단에 등록할 때 기입했던 '기타 설명 및 사양 정보'가 표출되지 않는 문제
- **개선 내용**:
  1. 카탈로그 데이터의 메모 영역(`item.note`) 필드가 실제 등록할 때는 정상 수집되고 있으나 상세 뷰(`CatalogDetailView.tsx`)에서는 렌더링 섹션이 누락된 사실을 확인.
  2. 상세 뷰의 닫기 버튼 직전 영역에 **기타 설명 및 사양 정보** 렌더링 카드를 새로 구축.
  3. `item.note` 가 비어있지 않고 값이 존재할 때만 표시(`없으면 없어야하고` 규칙 대응)되도록 연계하여 UI 완성.

## 10차 개선 작업
- **요청 사항**: 카탈로그 메인 리스트에서 상품 카드 이미지 하단의 정보 레이아웃 및 서식을 요청 양식에 맞춰 변경
- **개선 내용**:
  1. 기존의 카테고리 정보 노출 등을 제거하고 사용자가 지정한 아래 양식으로 개편:
     ```
     모델번호
     색상(W, P 등), 재질[순수금중량]
     스톤: 총스톤중량 g
     공임(1등): 기본공임(1등) + 스톤등급공임(1등)합산(세팅비 포함)
     [수량]스톤이름/설명
     ```
  2. `CatalogManager.tsx` 의 카드 꼬리 렌더링 구문을 전면 수정하여, 등급별 색상 동적 매핑, 재질 및 금중량, 스톤중량, 1등급 공임(기본공임 + 스톤 1등급 공임 합산 + 세팅비)을 실시간 산출하도록 수정.
  3. 등록된 스톤 목록을 돌며 `[수량]스톤이름/설명(비고)` 형식으로 각각 줄바꿈하여 가독성 높게 뿌려주도록 완성.

## 11차 개선 작업
- **요청 사항**: 상품 카드의 모델 번호 바로 아래에 한 줄 빈 줄 여백을 추가하고, 공임비와 스톤 공임비를 각각의 독립된 개별 라인으로 분리하여 출력하도록 변경
- **개선 내용**:
  1. 모델 번호 바로 아래의 정보 묶음 영역의 상단 마진(`marginTop: '10px'`)을 부여하여 개행된 공백 효과를 확실하게 적용.
  2. 기존 한 행에 합쳐져서 플러스 기호로 표기되던 구문을 분리하여 아래처럼 각각의 독립된 줄로 노출되도록 수정:
     - `공임(1등): {기본공임(1등)}원`
     - `스톤(1등): {스톤1등급공임합산}원`

## 12차 개선 작업
- **요청 사항**: 스톤 종류가 2개 이상일 때 상품 카드의 레이아웃 처리 기획
- **개선 내용**:
  1. 등록된 스톤이 여러 개일 때 카드를 아래로 길게 늘어뜨려 카드 간 높이가 불일치해지는 디자인 어긋남을 막기 위해, 스톤 정보를 쉼표(`,`)로 이어서 한 행으로 깔끔하게 정렬하도록 구현:
     - 예: `[1]SV/RD/물림비, [2]다이아`
  2. 이를 통해 스톤이 여러 개의 상황에서도 상품 카드 크기가 통일감 있고 프리미엄한 톤으로 유지되도록 보완 완료.

## 13차 개선 작업
- **요청 사항**: 카탈로그 등록 화면에서 불필요한 '알 공임비(개당)' 입력 필드 제거
- **개선 내용**:
  1. 스톤의 세부 단가 및 공임은 하단의 스톤 테이블(등급별 단가)의 각 행에서 개별 지정되어 합산 적용되므로, 상단의 단일 '알 공임비(개당)' 입력 폼은 사용자를 혼란스럽게 만들 수 있기에 UI에서 전격 삭제 조치.
  2. 이를 통해 기본재질, 단종여부가 위치한 영역이 더욱 깔끔하고 단순하게 정렬되도록 UX 개선 완료.

## 14차 개선 작업
- **요청 사항**: 카탈로그 등록 시 입력한 수동차감중량을 카탈로그 리스트의 스톤 중량에 합산하여 노출하도록 변경
- **개선 내용**:
  1. 기존 리스트 화면(`CatalogManager.tsx`)에서 '스톤:' 라인에 순수 스톤 마스터 데이터의 캐럿 중량만 단순 노출하던 부분을 보완.
  2. 스톤 실중량 합산에 **수동차감중량**(`manualDeductionWeight`)을 합한 총 차감중량(`totalDeductionWeight`)이 리스트의 `스톤:` 칸에 합산 및 노출되도록 연동 완료.

## 15차 개선 작업 (추가 요청)
- **요청 사항**: 카탈로그 상세화면에서도 수동차감 무게를 스톤 중량 항목에 포함하여 합산 노출하도록 변경
- **개선 내용**:
  1. 상세 페이지(`CatalogDetailView.tsx`)에서 '스톤:' 중량을 출력할 때 기존의 순수 스톤 중량(`totalStonesWeight`) 대신, 수동차감중량이 포함되어 있는 총 차감중량(`totalDeductionWeight`)을 모바일 뷰와 데스크톱 뷰 모두에 적용.
  2. 메인 리스트와 동일하게 상세 뷰에서도 수동차감중량이 포함된 통합 스톤 무게가 완벽하게 일치되어 나타나도록 보완 완료.
