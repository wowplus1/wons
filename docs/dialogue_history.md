*   **폼 격리 및 컴포넌트 분리**: 등록 폼 영역을 단독 구동 가능한 `StoneRegisterForm.tsx` 및 `CatalogRegisterForm.tsx` 파일로 신규 분리 생성.
*   **URL 쿼리 기반 팝업 라우팅 구현**: `App.tsx` 진입 시 `?popup=stone` 또는 `?popup=catalog`를 감지해 오직 해당 팝업 등록 폼만 전체 화면에 띄우도록 가상 라우팅 매핑.
*   **부모-자식 창 간 실시간 데이터 동기화 (postMessage)**: 
    *   팝업 창에서 등록을 끝내면 `window.opener.postMessage('db_update', '*')`를 전송하고 자동으로 윈도우가 `close()` 닫히도록 자동화.
    *   메인 부모 창은 이 메시지를 수신하여 깜빡임 없이 데이터베이스를 즉시 새로고침(`fetchDb()`)하도록 구성.
*   **본문 리스트 뷰 영역 확장**: 부모 창의 `StoneManager.tsx` 및 `CatalogManager.tsx` 에서 등록 폼이 제거됨에 따라, 목록 테이블과 격자 카드가 가로 100% 꽉 차게 표시되도록 가로 너비를 넓혀 데이터 밀도와 가독성을 향상.
*   **빌드 무결성 검증**: 미사용 Lucide 아이콘 정리 후 `npm run build` 최종 컴파일 성공 확인.
